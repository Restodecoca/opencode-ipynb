# Reference list for users who want real notebook execution (ipynb_run).
#
# The opencode-ipynb plugin does NOT install these automatically.
# It only detects them via `ipynb_doctor` and reports what is missing.
#
# Pick ONE of the following workflows. `uv` is preferred because it is
# fast, reproducible, and respects the user's existing Python install.
#
# Preferred (uv):
#   uv pip install nbformat nbclient jupyter_client ipykernel
#   python -m ipykernel install --user
#
# Alternative (pip into a venv):
#   python -m venv .venv
#   source .venv/bin/activate     # Windows: .venv\Scripts\activate
#   pip install nbformat nbclient jupyter_client ipykernel
#   python -m ipykernel install --user
#
# Alternative (conda):
#   conda install -c conda-forge nbformat nbclient jupyter_client ipykernel
#
# If you have multiple Python interpreters, point the plugin at the right
# one via either:
#   - opencode.json: { "ipynb": { "pythonPath": "C:/path/to/python.exe" } }
#   - env var:       OPENCODE_IPYNB_PYTHON=C:/path/to/python.exe
